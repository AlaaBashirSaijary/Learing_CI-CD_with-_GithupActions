🚀 دليل شرح GitHub Workflow (للمبتدئين)
هذا الملف يشرح بالتفصيل الممل كيف يعمل ملف الأتمتة (Workflow) الذي يقوم بمراقبة تفاعل المستخدمين مع "النجمة" (Star) الخاصة بالمستودع.

📄 الكود المستخدم
YAML
name: Star Event Workflow
on:
  watch:
    types: [started]
jobs:
  log-star-event:
    runs-on: ubuntu-latest
    steps:
      - name: checkout repository
        uses: actions/checkout@v3
      - name: log star event
        run: |
          echo "A user has starred or unstarred the repository."
          cat $GITHUB_EVENT_PATH
🔍 شرح الخطوات (Step-by-Step)
1️⃣ اسم الملف (name)
الوظيفة: هو "العنوان" الذي يظهر لك في واجهة GitHub Actions.

الفائدة: يساعدك في التمييز بين ملفات الأتمتة المختلفة (مثلاً: ملف للفحص، ملف للرفع، ملف للتنبيهات).

2️⃣ المُشغل (on)
الوظيفة: تحديد "الحدث" (Event) الذي يجعل الكود يبدأ بالعمل.

الشرح: هنا استخدمنا watch مع نوع started. في لغة GitHub، هذا يعني: "عندما يضغط شخص ما على زر الـ Star".

3️⃣ المهمة (jobs)
الوظيفة: الحاوية الكبيرة التي تضم الخطوات.

ملاحظة: الـ Workflow الواحد يمكن أن يحتوي على أكثر من مهمة (Job)، وكل مهمة تعمل بشكل مستقل.

4️⃣ بيئة التشغيل (runs-on)
الوظيفة: اختيار نظام التشغيل الذي سيعمل عليه الكود.

لماذا Ubuntu؟ لأنه نظام خفيف، سريع، ومجاني تماماً ضمن حدود GitHub Actions، ويوفر كل الأدوات البرمجية التي قد تحتاجها.

5️⃣ جلب الكود (actions/checkout)
الوظيفة: فتح اتصال بين "جهاز الكمبيوتر الافتراضي" و "المستودع الخاص بك".

الأهمية: بدون هذه الخطوة، لن يتمكن الجهاز الافتراضي من رؤية ملفات مشروعك أو التفاعل معها.

6️⃣ التنفيذ (run)
الوظيفة: كتابة أوامر برمجية مباشرة (Terminal Commands).

الأوامر المستخدمة:

echo: لطباعة جملة نصية بسيطة.

cat $GITHUB_EVENT_PATH: هذا أمر "سحري"، حيث يفتح ملفاً مؤقتاً يحتوي على تفاصيل الشخص الذي ضغط النجمة (اسمه، حسابه، وقت الضغط).
