🗺️ خارطة طريق مشروع الـ CI/CD
هذا المستودع ليس مجرد كود، بل هو رحلة تعليمية مقسمة إلى مراحل متسلسلة:

📍 المرحلة 1: التفاعل مع الأحداث (Events)
الملف: 01-star-event.yml

الهدف: فهم كيف يستجيب GitHub لأفعال المستخدم (مثل عمل Star).

النتيجة: رسالة ترحيبية آلية.

🛡️ المرحلة 2: الحارس الذكي (Continuous Integration)
الملف: 02-ci-testing.yml

الهدف: حماية الكود عبر الاختبارات التلقائية (Unit Tests).

النتيجة: علامة ✅ للنجاح أو ❌ للفشل لضمان جودة الكود.

📦 المرحلة 3: التغليف والترقيم (Artifacts & Versioning)
الملف: 03-artifacts-and-versioning.yml

الهدف: تحويل الكود السليم إلى "منتج نهائي" جاهز للتسليم.

النتيجة: إنشاء ملف مضغوط (Zip) يحمل رقم إصدار محدد (v1.1.0).
